### Codebase Patterns:

#### Architecture Pattern:
- **Static HTML + TypeScript modules**: Existing pages (index.html, contact.html) remain static
- **Module-based feature**: New animated background feature lives in src/ modules
- **Build system**: Vite bundles TypeScript modules for production

#### Naming Conventions:
- Files: kebab-case for HTML, camelCase for TypeScript
- Interfaces: PascalCase with descriptive names
- Functions: camelCase, verb-based names
- Constants: SCREAMING_SNAKE_CASE for config

#### Configuration Patterns:
- Centralized config with DEFAULT_CONFIG constant
- Type-safe interfaces for all configuration objects
- Separate config files per tool (tsconfig, vite.config, vitest.config)

#### Testing Patterns:
- Co-located tests: *.test.ts files next to source
- Vitest with jsdom environment
- Unit tests for pure functions and configuration
- Descriptive test names using BDD style

#### Build Flow:
1. TypeScript compilation (tsc --noEmit for typecheck)
2. Vite build (bundles and optimizes)
3. Output to dist/ directory
